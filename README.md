# TynysAlu — People's Air Network 🌬

> *"Know the air before you breathe."*

**TynysAlu** («дыши» на казахском) — концепт-прототип мобильного приложения для мониторинга качества воздуха в Алматы. Сеть датчиков и AI-камер в реальном времени собирает данные о загрязнении воздуха и превращает их в понятные людям рекомендации: стоит ли гулять, каким маршрутом ехать, когда закрыть окна.

Прототип полностью фронтендовый, собран в **Figma Make** и работает на моковых данных — бэкенда и реальных API нет.

---

## ✨ Возможности

- 📊 **Live AQI-дашборд** — стрелочный индикатор качества воздуха, PM2.5/PM10, температура, влажность, ветер
- 🗺 **Карта загрязнения** — станции на карте города с тепловыми зонами и фильтрами
- 📈 **Аналитика** — графики за 24 часа / 7 дней и AI-прогноз на 24 часа вперёд
- 🔔 **Умные алерты** — уведомления с объяснением причины и рекомендованным действием
- 🧠 **AI Insights** — развёрнутый текстовый анализ трендов (трафик, погода, промышленность) с указанием источников данных и % уверенности
- 📡 **Реестр станций** — статус онлайн/офлайн, камера, батарея, сигнал, прошивка
- 🛤 **Безопасный маршрут** — 3 варианта пути (безопасный / быстрый / сбалансированный) с оценкой AQI-экспозиции
- 👤 **Профиль** — сохранённые места, настройки алертов, язык (EN/KZ/RU), экстренные контакты

Ключевая идея сценария — двойной источник данных: **сенсоры + AI-камеры** (распознавание дыма/выбросов), что постоянно подчёркивается в интерфейсе как основа доверия к системе.

```
figma-make-app/

├── vite.config.ts

    └── screens/
        ├── SplashScreen.tsx     # заставка
        ├── HomeScreen.tsx       # главный экран / AQI-гейдж
        ├── MapScreen.tsx        # карта станций
        ├── AnalyticsScreen.tsx  # графики и прогноз
        ├── AlertsScreen.tsx     # лента уведомлений
        ├── InsightsScreen.tsx   # AI-анализ трендов
```

> Есть также вариант с объединением всего кода в один файл `App.tsx` — удобно, если нужно быстро вставить проект целиком без множества файлов.

## 🖥 Экраны

| Экран | Описание |
|---|---|
| **Splash** | Анимированная заставка с лого-хексагоном, загрузочной полосой и слоганом |
| **Home** | AQI-гейдж, AI-саммари состояния воздуха, метрики загрязнителей, ближайшая станция, быстрые ссылки |
| **Map** | Стилизованная карта города с маркерами станций, зонами загрязнения, фильтрами (все / онлайн / с камерой) |
| **Analytics** | Графики AQI и PM за 24ч / 7 дней, вкладка прогноза, детектор аномалий, таймлайн уверенности прогноза |
| **Alerts** | Уведомления с уровнями важности (critical / high / medium / low), причиной и рекомендованным действием |
| **Insights** | Развёрнутые карточки с AI-анализом: что произошло, почему, на основе каких данных |
| **Stations** | Реестр всех станций сети: статус, батарея, сигнал, прошивка, время синхронизации |
| **Route** | Три варианта маршрута с оценкой средней AQI-экспозиции и количеством "горячих точек" загрязнения |
| **Profile** | Сохранённые места, переключатели уведомлений, выбор языка, экстренные контакты |

## 🚀 Запуск проекта

```bash
# установка зависимостей
pnpm install    # или npm install / yarn install

# запуск dev-сервера
pnpm dev

# сборка
pnpm build

# просмотр production-сборки
pnpm preview
```

По умолчанию сервер разработки поднимается на порту `8443` (можно переопределить переменной окружения `PORT`).

## ⚠️ Ограничения прототипа

- Все данные — статичные моки в `mockData.ts`, реального подключения к сенсорам/камерам нет
- Карта — декоративная SVG-сетка, а не настоящая геокарта (без Google/Mapbox)
- AI-анализ и прогнозы — заранее написанные тексты, а не вывод настоящей модели
- Нет авторизации, бэкенда и сохранения состояния между сессиями
